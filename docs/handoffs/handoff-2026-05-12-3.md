## Key Decisions and Why

### Career Timeline got its own canvas view, not a tabbed swap
- **Decision:** Add a third action button on EndgameScreen that routes to a dedicated full-canvas timeline view. Reach it via button, return via Close.
- **Why:** The first proposal dropped `aspectRatio` so the timeline could scroll naturally. User vetoed: every other screen in the game is bounded by the same 1000×600 frame; breaking that for one screen is jarring.
- **Alternative:** Tabbed swap inside EndgameScreen. Rejected as visually noisier than a dedicated screen for the timeline payoff.

### Finale decision uses a synthetic interactable + finale-mode modal, not a new room
- **Decision:** `LOCKED_DOOR_INTERACTABLE` is a synthetic sprite (kind: 'object', art: 'locked-door', tags: ['finale'], weight: 0) injected into the December 2029 DecisionRoom. Bottom door routes to a hardcoded `FINALE_DECISION` constant rendered via `DecisionModal` with `finale: true`.
- **Why:** Reuses NPCModal/DecisionModal flow without adding finale content to the career pack. Keeps the locked door examinable (NPC modal pattern) without making it a real interactable in the pack content.
- **Alternative:** New `FinaleRoom` component. Rejected — too much surface for one beat.

### Title screen font: JetBrains Mono Bold, not Pixelify Sans
- **Decision:** `font-family: 'JetBrains Mono', monospace; font-weight: 700; letterSpacing: 0.02em; textShadow: '4px 4px 0 ${palette.accent}'`.
- **Why:** Pixelify Sans (nominated in the design doc) read as goofy the moment it rendered. The game's tone is contemplative-meets-Hitchhiker's, not arcade-pastiche.
- **Alternative:** Pixelify Sans (doc-nominated), display sans-serif. Both rejected as wrong register.

### Refresh-loses-init bug fixed via `store.subscribe`, not by widening `persistState` callers
- **Decision:** `store.subscribe(() => persistState(store.getState()))` in `store.ts`.
- **Why:** `persistState` was only fired from `useRoomTransition.exitRoom`, so any choice made before walking through the first door was in-memory only. Subscribe pattern is one line and covers all dispatches.
- **Alternative:** Manually dispatch persist after each init-flow action. Rejected as fragile (every new dispatch needs a paired persist).

### Begin Again reloads page; gameOver title acknowledge auto-resets state
- **Decision:** `EndgameScreen.handleReplay` calls `window.location.reload()` after reset dispatches. `App.tsx` title-screen key-press intercepts `gameOver === true` and dispatches the full reset chain + `clearPersistedState`.
- **Why:** Without reload, returning to title from a finished run preserved enough Redux state that pressing-any-key dropped the player on EndgameScreen instead of init flow ("I pressed start and got a summary").
- **Alternative:** Pure Redux reset (no reload). Rejected — the bug was that reset wasn't enough; persisted state was the culprit, and reload guarantees a clean slate.

### Tutorial overlay uses `position: absolute` inside DecisionRoom, not `position: fixed`
- **Decision:** Tutorial bubble is `position: absolute` inside a `position: relative` DecisionRoom container with `width: var(--canvas-display-width)`.
- **Why:** Fixed-position bubble was off-canvas on widescreens (canvas is centered with margin). Container-anchored positioning keeps the bubble glued to its referent regardless of viewport width.
- **Alternative:** Fixed positioning with manual viewport math. Rejected as fragile across breakpoints.

### Init flow buttons converted to outlined modal style (PR #49)
- **Decision:** Continue buttons match `DecisionModal` Continue / `CreditsScreen` Close / `EndgameScreen` action style: transparent bg, 1px ink border, uppercase 0.08em letterSpacing, `fontFamily: 'inherit'`, centered.
- **Why:** Init flow was using filled `palette.ink` buttons with `alignSelf: flex-end` — visually disconnected from in-game buttons.
- **Alternative:** Leave as-is. Rejected by user as "wrong style — doesn't match the other modals."

### Finale copy went through three drafts before landing
- **Decision:** Final option text: *"Hard to say. It mostly felt like a Tuesday." / "Tuesdays do most of the work." / "Sure. Probably. Mostly."* with landing line *"Hard to say. It mostly felt like a Tuesday. … Tuesdays do most of the work."*
- **Why:** First two drafts were too quippy (Blockbuster cards, GameBoy manuals). The breakthrough came when the user said *"take one more shot considering all the knowledge you have about this game"* — that explicit anchor produced the Tuesday line which echoes the `"Different month. Same coffee stain"` beat in the rewind-door status pool.
- **Alternative:** Earlier drafts (rejected as out-of-tone).

## Dead Ends

- **Dropping `aspectRatio` on EndgameScreen to allow timeline scrolling.** Vetoed by user — every screen must stay in the 1000×600 envelope. Lesson: never break the canvas frame for content overflow; build a dedicated view instead.
- **Pixelify Sans for title wordmark.** Rendered as goofy, wrong-tone. Lesson: design-doc font nominations aren't load-bearing; trust eyes over spec when register clashes.
- **Tutorial overlay with `position: fixed; inset: 0`.** Bubble landed off-canvas on widescreens because the canvas is centered with margin and the bubble's coordinate system was viewport-relative. Lesson: anchor overlays to the container they reference, not the viewport.
- **Initial finale copy with cultural references (Blockbuster, GameBoy manuals).** User rejected as cheesy. Lesson: this game's tone is contemplative-deadpan, not pop-quippy. Anchor finale-tier copy to existing in-game beats (the rewind-door status pool, the coffee-stain motif).
- **Trying to fix refresh-loses-init by sprinkling more `persistState` calls.** Considered briefly, abandoned — `store.subscribe` covers all dispatches in one line. Lesson: when a persistence bug spans many call sites, subscribe at the store, don't widen the caller surface.
- **Title-screen blink-pill (rounded background behind PRESS ANY KEY).** User said "blink was retro" — dropped the pill, kept the blink. Lesson: arcade INSERT COIN beat survives without modern chrome.
- **Pure Redux reset (no reload) for Begin Again.** Reset alone left persisted state on disk that re-hydrated on next mount, dropping the player back on EndgameScreen. Lesson: when localStorage is involved, reload is the honest reset.
