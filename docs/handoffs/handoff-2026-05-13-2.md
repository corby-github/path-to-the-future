## Key Decisions and Why

### Pack-aware class labels (manifest.classLabels) instead of removing the class system
**Decision:** Add a new optional `classLabels?: Partial<Record<string, ClassLabel>>` field on `Manifest` that lets a pack override any of the 8 universal `CLASSES.label` / `.role` values. ClassPicker reads `manifest.classLabels[id] ?? CLASSES[id]`; Hud reads same with `entryClasses.label` as last fallback.
**Why:** User flagged that the picker showed "Novice Initiate / Junior Adventurer / Vanguard Strategist" (RPG/SWE-flavored) to homeschool players. Two paths considered: rip out the class system entirely vs. override labels. Removal would touch `progress.classTier` state, HUD, endgame, design doc §14. Label override is ~45 min of localized change. SWE pack omits the field; falls through to CLASSES.label which is already SWE-flavored.
**Driver:** User chose "Pack-aware labels (recommended)" from a 4-option AskUserQuestion.

### 5+5 voice-checkpoint batch + defer 61 remaining homeschool icons
**Decision:** Author exactly 5 hp-* decision icons + 5 evt-hp-* event icons in PR #69, then a second 5+5 batch (20 total). Park the remaining 61 hp-* / evt-hp-* ids as `PlaceholderIcon` for now.
**Why:** The "homeschool icon ramp" item from this morning's handoff turned out to be 71 missing icons, not the ~5 the prior preview misleadingly suggested. Authoring all 71 would be 4–6 hours and burn the rest of the 5h budget. User explicitly chose "Highest-impact 20-25" from the AskUserQuestion. The polish-loop research artifact filed mid-session captures exactly this trade.
**Driver:** User. The morning handoff had pre-scoped the 5+5 batch; user confirmed mid-session.

### Hard-code 4 female homeschool NPC sprites for v1
**Decision:** Mother-in-law / spouse / co-op parent / neighbor all rendered as female. Created new `parent-*` art-token namespace to keep them separate from SWE `person-*` tokens.
**Why:** User explicitly chose this over (a) hard-coded mixed genders, (b) deferring until a gender-pick init phase ships. Matches existing copy register ("another mom at the park", "the neighbor mom", "a friend left the co-op"). Easy to flip individual sprites or add a runtime gender system later (logged as potential follow-up, not blocking).
**Alternative considered:** Adding a gender-pick init phase tonight (~1–4 hours depending on whether content also gets gendered pronouns). Rejected — would undo the recent spouse-`they` sweep if content goes Option B; even Option A's sprite-only version was deferred to keep PR #69 ship-ready tonight.
**Driver:** User. Explicit choice from AskUserQuestion.

### "We always homeschooled" framing fix
**Decision:** Rewrote 5 events + month-1 narrative to establish the player's family as the **steady continuous homeschoolers**, with the pandemic-era families cycling in temporarily and the rebound-era families returning to brick-and-mortar. Key lifted phrasings: *"the Zoom was the new part. The lessons weren't."* (zoom-cohort), *"three families who'd joined you in 2020"* (school-friends-return), *"You've been here long enough to know there'll be a sixth."* (evangelist-parent).
**Why:** User explicitly flagged that the pack was reading as if homeschool *started* during the pandemic — but they always homeschooled. The pandemic just means OTHER families started/stopped while the player's family kept doing what they'd always done. Anchors the pack's identity.
**Driver:** User during the homeschool review pass.

### +10 humor lift on heaviest decision flavors
**Decision:** 11 decision flavors rewritten with bolder comic kickers (e.g., *"Different version. Different fly."*, *"You'd been waiting for Bram to find his other shoe. Different kinds of waiting."*, *"the kitchen was, however, exceptionally well-organized that fall."*). Also 14 event-body lifts in the same register.
**Why:** Mid-evening, user named the existing tone as too somber — "boring and depressing" was the literal phrase. Used the user-authored tablet decision (Windows 98 / warehouse deal / confiscate) and "Research, allegedly" event as the target register. User asked for "+10 humor," not the previous "+1." Soft-death-adjacent decisions (dyslexia diagnosis, mental-health flag, aging-parent fall, graduation-frame) intentionally left alone — those stay weighty by design.
**Driver:** User explicit direction.

### Spouse-pronoun sweep (singular-they → "Your spouse" or sentence rewrite)
**Decision:** 10 spots rewritten to drop singular `they` for the spouse. Either replaced with "Your spouse" explicitly, or reframed to a different subject (e.g., `"They want to know if you want to talk about it tonight"` → `"Should you talk about it tonight"`).
**Why:** User flagged the singular-they reading as awkward — too easy to mis-read as plural. Plural-they usages (referring to kids, families, etc.) were left intact.
**Driver:** User.

## Dead Ends

### IconFortyTwo "head on hand" silhouette (commit `9b93edc`)
**What was tried:** A round-5 IconFortyTwo redraw on `feat/swe-icon-revisions` showing Deep Thought as a silhouette with a giant head resting on a bent-elbow arm (per a user reference image at that time). The commit was made AFTER PR #63 had already merged to main.
**Why it failed:** Branch was post-merge. The commit dangled on `feat/swe-icon-revisions` and never made it to main. PR #69 later did a fresh redraw based on a different user reference (line-art outlined wedge with feet), which is what eventually shipped.
**Lesson:** When iterating an icon AFTER its PR has merged, open a new PR — don't commit to a stale branch. The commit will get stranded and the iteration won't ship.

### Round-1 over-engineered three icons (IconRecruiterCall / IconCouch / IconHeartHands)
**What was tried:** First-pass redraws added detail to be "more literal" — crescent profile + grills + waves for the handset, fainting couch with button tufts + three legs, cupped palms with separated fingers + thumbs.
**Why it failed:** All three read as MORE confusing at 80×80 than the originals. User: *"IconRecruiterCall is worse! just make a profile of a handset, and be done."* *"IconCouch is worse — looks like a piano now."* *"IconHeartHands is worse! — just use a heart with a $ inside since its about donation."*
**Lesson:** Icon iteration converges by subtraction more than addition. Already captured in the afternoon session-process log; reinforced again with the evening's IconSlowTablet (spinner dropped) and IconFortyTwo (silhouette → outline).

### "80/81 real icons" mirage in homeschool preview
**What was tried:** Trusted the rendered-icon count in `docs/icons-previews/homeschool.html` as the coverage number — said earlier in the session "Items A–D done" including homeschool icon ramp.
**Why it failed:** The 80 rendered icons were SWE icons + universal-pool icons that the homeschool pack inherits. The actual hp-* / evt-hp-* coverage was 0. The "Missing icons" section at the bottom of the preview HTML was the real source of truth: 32 decisions + 39 events = 71 placeholders.
**Lesson:** Read the coverage gauge section of the preview HTML, not the rendered-icon count. The generator iterates the full registry, so cross-pack icons show in every pack's grid.

### Spinner on IconSlowTablet
**What was tried:** Added a 3/4-arc loading spinner inside the tablet screen to telegraph "slow / loading."
**Why it failed:** User flagged it as visual noise.
**Lesson:** "Tablet" alone reads fine at 80×80; the "too slow" framing lands from the modal context, not from in-icon iconography. Trust the surrounding text.
