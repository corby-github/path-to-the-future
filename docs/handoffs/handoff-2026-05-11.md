## Key Decisions and Why

### Voice/tone checkpoint cadence during content pass
**What:** Author a 5-decision + 5-event "starter batch," validate voice with user before scaling to the full 34/39 pool.
**Why:** Voice is load-bearing; writing 50+ entries in the wrong register is unrecoverable.
**Alternative considered:** Write everything then review at the end — faster but riskier.

### Stacker mechanic over generic twitch reaction
**What:** Reaction Sprint shipped as the **Stacker** — 5 blocks visible from start, bottom-up activation, spacebar to lock in the target column, soft-fail per miss.
**Why:** User wanted keyboard-only input (matches the rest of the game) and a more game-y mechanic than "click random target."
**Alternative considered:** Mouse-driven target-clicking (rejected as off-pattern with the keyboard-first input model).

### Endgame score: interpretable breakdown, not hidden number
**What:** `computeScore` returns a breakdown (`experience + savings/10 + wellbeing×25 − burnout×15 + relationship×20 + decisions×25`) displayed line-by-line on the recap.
**Why:** Players see why they scored what they did — fits the bittersweet/transparent voice.
**Alternative considered:** A single hidden balanced score (rejected — defeats the recap purpose).

### Day 13 phased into four sub-days
**What:** Original "Day 13 polish" slot rebalanced to **13a** (interactables system + 1 NPC + 1 object proof), **13b.1** (content fill + generator + multi-interactable), **13b.2** (sprite art + font architecture), **13b.3** (playability fixes from playtest). 13c (the original polish list) is still pending.
**Why:** §8b interactables were spec'd but never built; engine + content + visuals was a sprint-scale lift, not a polish slot. Splitting kept each PR reviewable.
**Alternative considered:** Cram into a single Day 13 PR (rejected as unreviewable); push to Day 14 (rejected — was scope-creep into a new build day).

### Font via `--font-game` CSS variable
**What:** Single CSS custom property in `:root` controls the game font. Body uses `var(--font-game)`. All inline `fontFamily` references collapsed to `'inherit'` except real monospace blocks (DevPanel, CodeReview's `<pre>`, Blackjack card faces).
**Why:** Future font swaps are one line. Currently set to JetBrains Mono.
**Alternative considered:** Per-component font props or class-based theming (rejected as bloat).

### Per-NPC pause (not global) + baseline network bump
**What:** Only the targeted NPC freezes on adjacency or NPC modal. Object interactions never freeze NPCs. Decision/event modals still freeze all (room-level pause beat). Talking to any NPC grants `+1 network` baseline.
**Why:** Playtest revealed the world felt dead during interactions, and Tier 1 flavor dialogues had no reward signal.

### Tiered multi-month feedback
**What:** Floater size + duration scales with magnitude (+1 mo small, +2–5 medium, +6+ large with "N months pass" wording). Status bar replaces random "life goes on" line with explicit "N months pass." for jumps ≥ 2.
**Why:** User: "when we jump months it's almost totally lost on the user — it can be a lot of time 'lost'."

## Dead Ends

- **Pixelify Sans modal-only.** Initial scope — only NPCModal used Pixelify Sans for the SNES homage. Created visible dissonance: NPC sprite (Inter-adjacent) vs. dialog (Pixelify). User flagged "the modals look separated from NPCs." Broadened Pixelify Sans across all UI → user flagged small text was illegible. Final answer: JetBrains Mono via `--font-game`. The lesson: scope retro-aesthetic decisions to *legibility constraints*, not just stylistic territory.

- **Original Stacker design (10 attempts, mouse).** First Day 11 spec was a 10-attempts twitch-reaction. User vetoed both mouse input and the unbounded-attempts framing. Rewrote mid-PR to "5 blocks visible from the start, spacebar to lock in column, bottom-up activation."

- **Single-NPC pause logic.** First implementation gated *all* NPC movement behind a single boolean ("npcShouldMoveRef"). Felt fine with one NPC; broke down with multiple — talking to one freezes all, looking at a printer freezes all. Replaced with three refs and a layered per-NPC check.

- **PR #19 merged early.** User clicked merge before two later commits (NPC movement + speed bump) were on the remote branch. The merge captured the branch as of the earlier state; the post-merge pushes orphaned two commits. **Recovery pattern noted:** `git checkout main && git pull && git checkout -b feat/day-13a-npc-movement && git cherry-pick <SHA1> <SHA2>`, open follow-up PR. Lesson: confirm push state before clicking merge during fast iteration.

- **TypewriterText with sync setState in effects.** Multiple iterations got hit by the `react-hooks/set-state-in-effect` lint rule. The rule isn't ceremony; it catches real anti-patterns. Final design: derived `complete` from `visible >= totalChars`, `skippedRef` for cancel signal, no synchronous setState inside the step useEffect body.
