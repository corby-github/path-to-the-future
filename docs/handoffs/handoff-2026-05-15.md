## Key Decisions and Why

- **NPC palette PR scope: NPCs only, don't split other `accent` uses.** The audit found 39 `palette.accent` references; 6 are NPC bodies. Once NPCs go yellow/green, the visual collision with doors goes away — no need to also split door / UI / minigame uses. Splitting further is its own design conversation (e.g. `doorAccent` token), worth filing as a follow-up if needed.
- **NPC accessory strokes stay on `palette.ink`.** Cups, glasses, clipboards, ties, watches, backpacks — not body parts. Keeping them on neutral ink preserves silhouette contrast against the new yellow/green bodies. Hair sites (designer tuft, Bram tuft, mother-in-law crown + bun, spouse long-hair frame, co-op-parent ponytail, neighbor bob) DID move to `palette.accent` because they're body color, and `palette.ink` (near-black) read too harsh against the new bodies and muted backgrounds. Alternative considered: match each body's `*Ink` token (uniform NPC outline) — rejected because accessories would blend more into the body color, losing silhouette clarity.
- **GoatCounter: dynamic script injection from `track.ts` rather than static `<script>` in `index.html`.** Cleaner — env-var-driven, dev never loads it, no Vite `index.html` template hacks. Alternative considered: static script in HTML with Vite env-var substitution — more boilerplate, less control.
- **Custom event params encoded as query-string suffix on the path** (e.g. `game_started?career=software-engineering&class=skilled`). GoatCounter's `count()` API doesn't natively support key-value params on events; query-string encoding groups cleanly in the dashboard. Alternative considered: multiple event sends per data point — would inflate event count.
- **`/init/kid-names` slug added to §24 spec** (originally missing). v2.0.14 introduced KidNamesEntry as a real init-flow phase but §24 was never updated. Added as a one-line addendum to the slug table.
- **CreditsScreen mode='replay' fires `/restart` pageview, not `/credits`.** Two-step funnel: player reaches the begin-again confirm screen → `/restart` pageview; player clicks "Yes, begin again" → `restart_confirmed` event. Cleaner than firing both from a single trigger.
- **Easy-tier scope: re-tag maze + author 2 new templates.** Maze leaves the simple pool entirely; easy gets `maze` / `s-curve` / `switchback`. Alternatives considered: leave maze as simple + author all-new easy templates (more authoring), or re-tag maze alone (single-template easy pool, repetitive). Three-template easy pool is enough variety for v1.
- **Medium-tier collision: detect-only, NOT in static blocker list.** Moving obstacles are detected separately from the player's static-collision system; player can walk through but takes a hit + knockback. Avoids the "stuck inside obstacle" bug that would occur if a moving obstacle moved into a player while the obstacle was in the static collision list. Matches §4 spec wording: "throws the player back, takes some health."
- **Knockback direction: west (toward spawn), not perpendicular to obstacle motion.** Deterministic, regardless of which obstacle hit the player or which direction it was moving. Reads as "the wall pushed me back the way I came." Alternative considered: away from obstacle center — more physically realistic but felt arbitrary in playtest reasoning.
- **`XP_TIER_BONUS_UNTOUCHED = 100`.** Sized between `XP_MINIGAME_PARTIAL` (100) and `XP_MINIGAME_WIN` (250). A clean traversal of a hazard room reads as a meaningful but not-overwhelming reward.

## Dead Ends

- **`react-hooks/immutability` rule blocked the standard ref-mirror pattern in DecisionRoom.** Tried multiple variations of `useRef` + `useEffect` to track the latest `movingObstacleRects` (custom-hook return) and `player.setPosition` (custom-hook return property): empty-list sentinel + useEffect with no deps, sentinel + useEffect with deps, same-expression initial value + useEffect with deps. All three failed lint with `Modifying a value previously passed as an argument to a hook is not allowed`. Pong.tsx uses the EXACT same pattern (`scoreRef` / `paddleYRef` etc.) and lints clean — the rule treats values flowing from a custom-hook return differently from values flowing from a local `useState`. Couldn't crack the subtle compiler-aware difference. Resolved with `eslint-disable-next-line react-hooks/immutability` on those two specific lines, with a comment pointing at the Pong precedent. Worth revisiting if the rule drops the false-positive or if a cleaner pattern emerges.
- **`performance.now()` inside `useRef(performance.now())`** in `useMovingObstacles.ts` triggered "Cannot call impure function during render." Fixed by switching to `useState(() => performance.now())` — useState's lazy initializer is the sanctioned spot for impure calls. The clock anchor still re-anchors per room remount via a useEffect on the obstacle-list identity.
- **GoatCounter "not counting because of: localhost" warning** initially looked like the wiring was broken. Turned out it's GoatCounter's default localhost protection. Fix wasn't a bug fix — it was an env-flag addition (`VITE_ANALYTICS_ALLOW_LOCAL`) that, when set in `.env.production.local`, injects `data-goatcounter-settings='{"allow_local":true}'` on the script tag. Lesson: that warning is *proof the wiring works* (script loaded, `count()` called, GoatCounter received it).
- **Day-5 afternoon time-log appeared unclosed at session start.** I almost edited the time-log to backfill the missing end row — turned out the user had committed it in another session. They just needed to `git pull`. Lesson: in two-thread workflows, check `git pull` before assuming local state desyncs are real bugs.
- **Punch commit landed on `feat/medium-tier-templates` instead of `main`.** Ran `git add docs/logs/time-log.md && git commit && git push origin main` — but `git push origin main` from a feature branch does nothing useful (no commits to push there). Fixed with `git cherry-pick` onto main; the duplicate commit on the feature branch will dedupe at PR merge time (same diff content). Lesson: the punch skill's commit step assumes you're on main; check `git branch --show-current` before committing.
